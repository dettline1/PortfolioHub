## Детальное описание

### 📁 Корневая директория

#### Основные файлы приложения

**`app.py`**
- Главный файл Flask приложения
- Маршруты (routes): `/`, `/api/projects`, `/generate-sitemap`, `/generate-readme`
- Функции загрузки и парсинга проектов
- Логика поиска и фильтрации

**`config.py`**
- Конфигурация приложения
- Информация об авторе
- Настройки сайта (URL, название)
- SECRET_KEY для Flask

**`requirements.txt`**
- Список Python зависимостей
- Flask==3.0.0
- Jinja2==3.1.2
- Werkzeug==3.0.1
- watchdog==3.0.0

### 📁 projects/

Папка с проектами. Каждый проект - это подпапка с файлом `info.json`.

#### Структура info.json:

```json
{
  "name": "Название проекта",
  "description": "Описание проекта (10-500 символов)",
  "tags": ["tag1", "tag2", "tag3"],
  "link": "https://github.com/username/repo"
}
```

#### Обязательные поля:
- `name` (string) - название проекта
- `description` (string) - описание
- `tags` (array) - список технологий/тегов
- `link` (string) - ссылка на GitHub

#### Примеры проектов:
- `ai-notes-bot/` - Телеграм-бот с AI
- `web-scraper/` - Веб-скрапер
- `portfolio-generator/` - Генератор портфолио
- `data-analyzer/` - Анализатор данных
- `task-automation/` - Автоматизация задач

## 📊 Поток данных

1. **Запуск приложения:**
   - `app.py` загружается
   - Импортируется `config.py`

2. **Обработка запроса:**
   - Пользователь открывает `/`
   - `load_projects()` сканирует папку `projects/`
   - Читает все `info.json` файлы
   - Парсит JSON в Python словари

3. **Фильтрация:**
   - Применяется поиск (если есть `?search=`)
   - Применяется фильтр по тегам (если есть `?tag=`)

4. **Рендеринг:**
   - Данные передаются в `templates/index.html`
   - Jinja2 рендерит HTML
   - Подключается `static/style.css`
   - Отправляется пользователю
